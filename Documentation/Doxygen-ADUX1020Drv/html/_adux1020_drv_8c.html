§ FIFO_WATERMARK

#define FIFO_WATERMARK   ((0x04U) << 8U)

Define FIFO_WATERMARK value in the size of WORD, the fifo interrupt occure after reaching this level. The value between 0x00U to 0x0FU

§ INT_ENA

#define INT_ENA   ((0x01U) << 2U)

Macro definition for Interrupt Enable Interrupt pin is enabled Define INT_ENA as 0 to disable it

§ INT_POL

#define INT_POL   ((0x01U) << 5U)

Interrupt polarity is set as active low Define INT_POL as 0 for active high, 1 for active low

#define INTERRUPT_ENABLE

Define the INTERRUPT_ENABLE macro to enable the Interrupt related macro's

§ Adux1020DrvReadFifoData()

int16_t Adux1020DrvReadFifoData ( uint8_t *  pnData,
uint16_t  nDataSetSize 
)

Read data out from Adux FIFO.

Parameters
*pnData- output buffer pointer.
nDataSetSize- DataSet Size to be get
Returns
int16_t A 16-bit integer: 0 - success; < 0 - failure

The first argument to the function is the the register (FIFO) address of the Adux device from where the data is to be read. The 2nd argument is the pointer to the buffer of received data. The size of this buffer should be equal to the number of data requested. The 3rd argument is the size of requested data in bytes. ADUX_I2C_TxRx() should be implemented in such a way that it transmits the register address from the first argument and receives the data specified by the address in the second argument. The received data will be of size nDataSetSize.

§ Adux1020DrvRegWrite()

int16_t Adux1020DrvRegWrite ( uint16_t  nAddr,
uint16_t  nRegValue 
)

Synchronous register write to the Adux.

Parameters
nAddr- 16-bit register address
nRegValue- 16-bit register data value
Returns
int16_t A 16-bit integer: 0 - success; < 0 - failure

The 1st argument to the function Adux_I2C_Transmit is the pointer to the buffer of the size of three bytes. The first byte is the register address of the Adux device followed by the 16 bits data value to be written to the device register. The 2nd argument is the size of the buffer in bytes (3 bytes). ADUX_I2C_Transmit() should be implemented in such a way that it transmits the data from anI2cData buffer of size specified in the second argument.
